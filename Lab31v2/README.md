* **lab31v2.Tests** — тестовий проєкт, налаштований з використанням фреймворку xUnit та бібліотеки Moq.

## 3. Опис бізнес-логіки
Основним компонентом є клас `UserService`, який керує логікою авторизації та реєстрації користувачів. Клас побудований з використанням патерну Dependency Injection та отримує свої залежності через конструктор:
* `IUserRepository` — абстракція для доступу до даних користувачів (додавання, оновлення, пошук).
* `IPasswordHasher` — абстракція для криптографічної обробки паролів (хешування та верифікація).

Реалізовані методи сервісу:
* **Register** — реєстрація нового користувача. Перевіряє унікальність імені та хешує пароль перед збереженням.
* **Login** — авторизація. Перевіряє наявність користувача та валідність пароля.
* **ChangePassword** — зміна пароля після підтвердження старого.

---

## 4. Стратегія тестування
Для перевірки роботи `UserService` в ізоляції від реальної бази даних застосовано бібліотеку Moq. Написано 8 модульних тестів, які покривають як позитивні, так і негативні сценарії.

Основні інструменти Moq, що були використані:
* `Mock<T>` — створення імітацій залежностей.
* `Setup(...).Returns(...)` — визначення поведінки мок-об'єктів (наприклад, повернення існуючого користувача або результату перевірки пароля).
* `Verify(...)` — перевірка факту виклику методів залежностей (наприклад, гарантія того, що метод `Add` не викликається при помилці реєстрації).

Покриті тестові випадки:
1. Успішна реєстрація нового користувача.
2. Відмова у реєстрації, якщо користувач вже існує.
3. Відмова у реєстрації при передачі порожніх даних.
4. Успішна авторизація з правильними обліковими даними.
5. Відмова в авторизації при хибному паролі.
6. Відмова в авторизації неіснуючого користувача.
7. Успішна зміна пароля.
8. Відмова у зміні пароля при неправильному введенні старого пароля.
