## Структура проєкту

Рішення (Solution) складається з двох проєктів:

---

## Реалізована бізнес-логіка (MathHelper)

Клас `MathHelper` містить реалізацію наступних математичних алгоритмів:
- `IsPrime(int number)` — перевірка, чи є число простим.
- `Factorial(int n)` — обчислення факторіалу числа (з перевіркою на від'ємні значення).
- `Fibonacci(int n)` — знаходження $n$-го числа Фібоначчі.
- `GCD(int a, int b)` — знаходження найбільшого спільного дільника (НСД) за алгоритмом Евкліда.

---

## Тестування

Для перевірки працездатності коду написано 10 юніт-тестів, які використовують атрибути `[Fact]` для поодиноких перевірок та `[Theory]` з `[InlineData]` для параметризованих тестів. 

Покриті сценарії:
- **Успішні сценарії:** коректне обчислення математичних операцій.
- **Помилки (Exceptions):** генерація `ArgumentException` при передачі від'ємних значень для факторіалу та Фібоначчі.
- **Крайові випадки (Edge cases):** передача нулів, від'ємних чисел для НСД, перші числа послідовностей.
